티켓팅 전체 플로우

각 단계를 클릭하면 상세 내용을 볼 수 있습니다

1
대기열 진입
Redis ZSET · score=timestamp ›
POST /queue/matches/{matchId}/enter 요청 → Gateway에서 JWT 검증 후 X-User-Id 주입 → Redis ZADD queue:wait:{matchId} (score=타임스탬프) → 좌석 선호도 seat:preference:{m}:{u} 저장 (TTL 900초) → 클라이언트에 WAITING 상태 + 순위 반환
2
폴링 (동적 간격)
rank ≤ 100 → 1.5s / ≤ 1000 → 3s / 그 외 → 5s ›
클라이언트가 동적 간격으로 GET /queue/status 폴링 → 스케줄러가 상위 사용자를 READY로 승격 → queue:ready:{m}:{u} 에 Admission Token 저장 (TTL 30초) → 클라이언트에 admissionToken 반환
3
Seat 진입
Admission Token 일회용 즉시 소멸 → 대기열 우회 원천 차단 ›
POST /seat/enter (admissionToken) → Redis queue:ready GET + DEL (토큰 즉시 소멸, 재사용 불가) → seat:preference 로드 → seat:session 생성 + ZADD seat:active → 클라이언트에 seat_session 쿠키 발급
4
추천 블록 조회
취향 점수 최대 70점 ›
GET /recommendations/blocks → 온보딩에서 선택한 선호 블록(최대 10개)만 탐색 → 각 블록에서 AVAILABLE MatchSeat 조회 → row별 그룹화 → 연속 구간 추출 → N연석 묶음 수 계산 (구간길이 − N + 1) → 뷰포인트(최대 30점) + 응원구단(25점) + 근접 선호(15점) 취향 점수 산출 → 연석 수 차이 ≤ 10개 → 취향 점수 우선 정렬 → 추천 블록 카드 리스트 반환
5
좌석 배정 + Hold
Redis 분산 락 SETNX (TTL 5s) · Hold 5분 ›
POST /blocks/{blockId}/assign → SETNX block_lock:{blockId} 획득 → 진짜 연석 탐색: 같은 row 내 templateColNo 연속 N개, 앞열·통로 가까운 순 정렬 → 연석 없으면 준연석 fallback: 인접 2 row에 걸쳐 N석 분배, 수평 겹침(overlap) > 0 필수 → match_seats → BLOCKED + seat_holds 생성 (TTL 5분) → 락 해제 → 배정 좌석 + holdExpiresAt 반환
6
주문 + 결제
Hold 검증 → 주문 생성 → Mock 결제 ›
GET /orders/sheet (Hold 검증 + 가격 계산) → 주문서 반환 → POST /orders → INSERT orders (PAYMENT_PENDING) → POST /orders/{id}/payment → 토스페이/카카오페이: 즉시 COMPLETED / 가상계좌: PENDING (입금기한 3일) → 결제 완료 응답
